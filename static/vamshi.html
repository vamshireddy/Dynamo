    • Projects

    Compiler for COOL: Compiler written in C++ for Object oriented language COOL. It has Lexical analyser, Parser, Semantic analyser and Code generation phases. I have used flex and bison for lexer and parser respectively. Currently, I am working on semantic analyzer and code generation. It is expected to be done by December. This is built for MIPS architecture.

    IO Multiplexed FTP server: A Scalabe FTP server is designed in C with IO multiplexing and multithreading to handle a large number of clients per thread. It uses epoll system call to efficiently handle data and control socket descriptors with minimum number of threads. This server has less scheduling and memory overhead when compared to other native Linux FTP servers.

    Netflow Exporter for Linux: Netflow flow monitoring tool is written using libpcap in C for Linux. I have developed a flow cache to store statistics of packets flowing through network interfaces. (RFC). This can be used for packet monitoring and data analytics of network statistics.

    Mitigating ARP Spoofing Attacks: ARP spoofing attack mitigation module is implemented on POX-SDN controller. It is written in Python. Controller maintains state information about the allocated MAC, IP pairs by ARP and DHCP packet inspection. Controller reacts and mitigates ARP attacks by installing flow entries on the switches. This work is published in ICCTA, Alexandria, 2015

    Centralized Logging Server: A Centralized, Scalable and Reliable UDP based logging server developed to log simultaneous messages coming from large number of clients.

    Mighty Conferencing System: A video streaming server and client software written in C. Server helps the clients to initiate P2P connections between them. It has features such as Call initiation, Call termination, Busy Notification, etc.

    L3 Router cum Firewall for POX controller: A router with firewall module is developed in Python for SDN controller POX. It uses open-flow rules to instruct the switches. This was developed as a part of SDN course and simulated on Mininet.

    Remote Lab Monitoring System: Developed an end to end system to help monitoring the lab computers of my college. The system is written in C# and ASP.NET. It has a multithreaded server running with services, which makes connections to lab machines for monitoring. Live screenshot stream, Remote shutdown, Process monitoring are some of its features.

    ARP Cache Poisoning Tool: Developed using raw sockets in C. It can perform ARP request and reply based spoofing attacks.

    Ping Tool for UNIX: This is a clone of the native Ping application present in Unix systems. It is developed using raw sockets in C. It is used for ping, which displays RTT values and handles timeouts, etc.

    Distributed Classroom Quizzing System: A Distributed classroom system is developed in Java, which allows clients to form groups and perform inter group quizzes in a WLAN setup. A specialized reliable protocol above UDP with sequence numbers and retransmissions is developed as a part of this project. Leader election, group formation and failure detection are performed on peers connected to the system. This system can advance inspite of various failures in WLANs.

    Static Router - CS144 (Stanford) : Layer 3 Router is developed in C, which consists of modules such as ARP cache, ICMP packet handling, TCP/UDP handling, etc. This is a complete router except dynamic routing protocols. This should handle all of the ICMP errors and the applications that use them such as Ping, Traceroute, etc. It takes static routing table entries and enforces routing policies. Topology is simulated in Mininet.
